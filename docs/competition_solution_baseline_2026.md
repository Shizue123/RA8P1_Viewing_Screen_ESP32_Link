# 2026 瑞萨赛道技术方案基线

## 作品建议名称

智境方舟 - 基于 RA8P1 的可插拔环境感知与 Agent 协同控制终端

## 赛题匹配

作品以 RA8P1 为主控，主要功能代码运行在瑞萨平台，兼顾三个方向：

- 物联网：温湿度、光照等多源感知，MQTT 状态同步和远程运维。
- 嵌入式人工智能：云端 Agent 理解自然语言，RA8P1 执行受限规则；后续可将异常检测模型部署到 RA8P1。
- 大工业/工业 HMI：本地触摸屏、标准口、状态诊断、执行反馈和离线可运行控制。

## 落地场景

首选场景为温室育苗柜、实验室样品柜或小型设备间。

系统通过 AHT20 获取温湿度，通过 BH1750 获取光照；RA8P1 在本地完成采样、状态判断、触摸显示和 SG90 风门/遮光机构控制。网络可用时，ESP32-S3 将状态同步到云端，用户可用自然语言查询环境或生成控制规则。网络断开时，已下发规则仍由 RA8P1 独立执行。

## 核心架构

```text
AHT20 / BH1750
       |
PCA9548A + I2C-1
       |
RA8P1
  |- 驱动与诊断
  |- 设备注册/能力模型
  |- LVGL + FT6336 HMI
  |- rule_program 状态机
  |- SG90 / 告警执行
       |
115200 UART
       |
ESP32-S3
  |- Wi-Fi / MQTT / NTP
       |
Hermes + DeepSeek + Web/QQ
```

## 已实现的软件模块

1. ILI9488 显示、FT6336 触摸和 320 x 480 竖屏 LVGL UI。
2. PCA9548A 复用总线、AHT20 与 BH1750 驱动及细粒度故障诊断。
3. `platform_ports + device_registry + telemetry.samples` 统一状态模型。
4. 动态模块列表、模块详情、标准口和 Wi-Fi 配网页面。
5. SG90 0-180 度控制、受限动作序列、冷却时间和执行状态回传。
6. ESP32-S3 UART/Wi-Fi/MQTT/NTP 桥接。
7. 云端 Hermes/DeepSeek 自然语言理解、知识约束、参数校验和 MQTT 下发。
8. `deploy_ack / execution_state / status / telemetry` 闭环证据。

## 可量化指标

| 指标 | 当前值或验收口径 |
|---|---|
| 屏幕分辨率 | 320 x 480 |
| RA8P1-ESP32 串口 | 115200 bit/s，8N1 |
| I2C 模块 | PCA9548A + AHT20 + BH1750 |
| 感知能力 | 温度、湿度、光照 |
| SG90 控制范围 | 0-180 度 |
| SG90 控制周期 | 20 ms |
| 云端状态周期 | 15 s，事件变化可触发提前上报 |
| 上电数据可用验收 | 20 s 内出现有效温湿度和光照 |
| 环境变化响应验收 | 15 s 内屏幕或网页出现新值 |
| 动作状态 | ARMED / TRIGGERED / DONE |

以上时间为工程验收口径，不替代后续使用仪器完成的正式统计。

## 主要创新点

1. 单物理标准口通过复用器承载多模块，并保持“物理总线、挂载设备、能力样本”三层事实一致。
2. 屏幕不是写死设备页面，而是从设备注册表动态生成入口。
3. 大模型不直接控制 GPIO；自然语言先转成白名单 DSL，再由 RA8P1 状态机执行。
4. 屏幕、网页和对话端共用新鲜度与证据规则，缺少实时数据时明确报告不可用。
5. 云端失联不影响 RA8P1 已部署规则继续运行。

## 当前限制

- SG90 是开环执行器，尚无位置反馈。
- 当前异常检测文档与采样方案已准备，正式 TinyML 模型尚未作为已完成能力申报。
- 蜂鸣能力已进入统一能力模型，答辩前需以最终接线和实机复测结果决定是否列为完成项。
- 正式性能报告仍需补充多轮冷启动、网络断开、传感器拔插和控制响应统计。
