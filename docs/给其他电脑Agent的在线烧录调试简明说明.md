# 给其他电脑 Agent 的在线烧录调试简明说明

更新时间：2026-06-03

## 1. 这份文档是干什么的

这份文档是写给“另一台电脑上的 agent”看的。

目标不是讲复杂原理，而是让它能用简单、稳定、可重复的方法完成这件事：

```text
电脑连上板子
-> 编译代码
-> 烧录进板子
-> 让程序跑起来
-> 在需要时停下来调试
-> 判断到底是代码问题、连接问题，还是工具问题
```

## 2. 先用一句话讲明白

所谓“通过数据线给板子在线烧录和调试”，本质上不是 USB 线自己会烧录。

真正发生的是：

```text
电脑
-> USB 数据线
-> 调试器（板载 J-Link 或外接 J-Link）
-> SWD 调试线
-> MCU 芯片
```

也就是说：

- USB 负责把电脑连到调试器
- J-Link 负责把电脑命令翻译成芯片能听懂的调试命令
- SWD 负责把这些命令真正送进 MCU

所以能在线烧录、在线调试，关键不是“有一根线”，而是“这根线后面接着调试器”。

## 3. Agent 只要记住这 3 件事

### 3.1 编译

先把源码编译成固件文件。

常见产物：

- `.elf`
- `.bin`
- `.hex`
- `.srec`

没有这些产物，就还没到烧录那一步。

### 3.2 烧录

烧录就是让调试器把固件写进 MCU 的 Flash。

常见工具：

- `J-Link Commander`
- `OpenOCD`
- `pyOCD`
- 芯片厂商自带下载工具

### 3.3 调试

调试就是在程序运行时，通过调试器控制 MCU。

常见能力：

- 复位
- 暂停
- 继续运行
- 单步执行
- 下断点
- 看变量
- 看寄存器
- 读内存
- 写内存

## 4. 给 Agent 的最简单工作流程

不要一上来就乱点 IDE。

按下面顺序做，最稳。

### 第一步：确认电脑已经识别到调试器

先确认：

- 数据线插好了
- 板子上电了
- 调试器被电脑识别了

如果电脑根本没识别到 J-Link，那后面所有“烧录失败”都只是表象。

### 第二步：编译工程

目标：

- 成功生成固件文件

通过标准：

- 没有编译报错
- 生成了 `.elf` 或其他下载产物

### 第三步：连接目标芯片

目标：

- 调试器真的能找到目标 MCU

通过标准：

- 工具能识别目标芯片型号
- 连接接口正确，一般是 `SWD`

如果这一步失败，优先怀疑：

- 线没接对
- 板子没上电
- 芯片型号填错
- J-Link 版本不对

### 第四步：下载固件

目标：

- 把刚编好的程序写进板子的 Flash

通过标准：

- 下载工具明确返回成功
- 没有 `flash download error`

### 第五步：运行程序

目标：

- 让 MCU 从新程序启动

常见动作：

- `reset`
- `go`
- `run`

### 第六步：如果要调试，就停在断点或 main

目标：

- 能停住
- 能单步
- 能看变量

如果程序能烧进去，但完全停不住，通常就不是“编译问题”，而是“调试链路问题”。

## 5. 最容易理解的角色分工

可以把整套东西想成 4 个角色：

- `电脑`：发命令、编译代码、显示调试结果
- `USB`：把电脑连到调试器
- `J-Link`：翻译调试命令
- `MCU`：真正被烧录、被调试的目标芯片

其中最关键的是 `J-Link`。

没有它，电脑通常不能直接靠一根普通 USB 线去控制 MCU 内部运行。

## 6. 为什么一根线能同时做“烧录”和“调试”

因为烧录和调试本来就走的是同一条底层通道。

区别只在于：

- 烧录：主要是在写 Flash
- 调试：主要是在控 CPU 和读写内存

所以在 IDE 里看起来像两个按钮，底层其实是同一套连接。

## 7. 别的电脑上的 Agent 应该优先怎么做

推荐优先级：

1. 先用命令行工具确认调试器和芯片能连上
2. 再编译
3. 再烧录
4. 最后才开 IDE 调试界面

原因很简单：

- 命令行更稳定
- 更容易复现
- 更适合 agent 自动执行
- 出问题时更容易分层定位

## 8. Agent 的判断规则

Agent 不要把所有问题都归结成“代码错了”。

按下面方法判断：

- 编译失败：代码或工具链问题
- 连不上目标：连接、供电、驱动、调试器问题
- 能连上但下不进去：烧录配置、芯片型号、Flash 访问问题
- 能烧进去但程序不跑：固件逻辑、时钟、启动流程问题
- 能跑但断点不停：调试配置问题

## 9. 给其他 Agent 的简短提示词

下面这段话可以直接发给别的 agent：

```text
你现在负责一块 MCU 板子的在线编译、烧录和调试。

请按这个顺序工作：

1. 先确认电脑是否识别到调试器
2. 再确认调试器是否能连上目标芯片
3. 再编译工程并确认产物生成
4. 再执行烧录，不要直接假设 IDE 已经成功下载
5. 如果需要调试，再停在 main 或断点处检查变量和寄存器

请始终把问题分成四层排查：
1. 电脑和工具层
2. 调试器连接层
3. 固件编译与烧录层
4. 程序运行层

不要把所有问题都当成代码问题。
每完成一步，都要给出明确证据，再进入下一步。
```

## 10. 一句话总结

“电脑通过数据线给板子在线烧录和调试”这件事，简单说就是：

`电脑用 USB 连到 J-Link，J-Link 再通过 SWD 控制 MCU，所以既能写程序，也能停程序、看变量、单步调试。`
