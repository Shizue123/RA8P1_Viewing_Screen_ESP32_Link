# 统一状态载荷示例（2026-06-16）

本文档用于给网页前端、网页后端和其他集成人员提供当前固件侧的统一状态载荷参考。

当前方向是：

- `status` 优先提供 `ports`
- `telemetry` 优先提供 `samples`
- 旧字段继续兼容保留

## 消费要求

网页前端和后端后续应优先按以下规则消费：

1. 设备状态页优先读取 `payload.ports`
2. 实时数值优先读取 `payload.samples`
3. 规则入口可见性由 `capabilities[].id` 决定
4. 旧字段仅作为兼容兜底，不再作为新增功能主路径

## 兼容说明

当前固件和桥接层仍保留旧字段，例如：

- `payload.aht20`
- `payload.i2c`
- `payload.hardware_list`
- `payload.buses`

这些字段是兼容层，不应作为后续新增页面和新规则入口的主依赖。
